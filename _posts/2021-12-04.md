As I like to see the future of RHEL, I have installed Centos 9 Streams on one of my home servers. One expected problem at hte time of writting is thier are no third party repositories, not even EPEL. This is not really a problem as we can run all the services in a container.



## Podman to the Rescuce.
Podman is a daemonless, and rootless, container engine. Podman cam be run as root or non-root user. Podman is compatible with Docker amd Dockerfiles. You can run Docker Compose files with a little setup.
Some other advataged of podman is the ability to generate systemd service files or kuberneties Pod yaml files. Podman can be greate at converting your docker or docker-compose file to kuberneties files.

## Deploying Jupyter Notebook with podman

Starting with something simple, we will deploy Jupyter Notebook.

Podman will automatically create a volume for the image, but we create the volume that is human reaable.

```bash:
podman volume create --name jupyter-work
```
 You can also point to a directory on your computer, like /home/user/work. Replace jupyter-work with /home/user/work:/home/jovyan/work:Z

Normally we would simply run

```bash:
podman container run -d \
    --name jupyter \
    -p 6000:8888 \
    -v jupyter-work:/home/jovyan/work:Z \
    jupyter/scipy-notebook
```
But than we have to run podman logs jupyter and copy the token from tne terminal, kind of clunky. What we can do is create our own tome or disable it completly. C

```bash:
podman container run -d \
    --name jupyter \
    -p 6000:8888 \
    -v jupyter-work:/home/jovyan/work:Z \
    jupyter/scipy-notebook start-notebook.sh --NotebookApp.token=''
```


Podman will create a image named jupyter. It will be accessiable from port 6000. Files will be saved in ~/.local/share/containers/storage/jupyter-work/

If you are deploying on a system running SELinux, we need to add a z or Z to the end of the volume. If the volume will be accessed by more than one container use z, otherwise use Z.

We can check if jupyter was setup correctly by running:
```Dockerfile:
podman container ls
```
